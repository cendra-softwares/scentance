# PhonePe Payment Gateway - Changes and Updates (2025-2026)

## Major Changes in 2025

### 1. New SDK Architecture (v2 SDK)
- **Package**: `@phonepe-pg/pg-sdk-node` (v2.0.5+)
- **Old**: Direct API calls with X-VERIFY checksums
- **New**: OAuth-based authentication with SDK abstraction
- **Benefit**: Simplified integration, automatic token management

### 2. OAuth 2.0 Authentication (v2 API)
- Replaced X-VERIFY for standard checkout flows
- Uses client credentials grant type
- Token endpoint: `/apis/pg-sandbox/v1/oauth/token` (UAT)
- Token endpoint: `/apis/identity-manager/v1/oauth/token` (Production)

### 3. Standardized Client Initialization
```javascript
// New approach (2025)
const client = StandardCheckoutClient.getInstance(
  clientId,
  clientSecret,
  clientVersion,  // Integer
  Env.SANDBOX     // or Env.PRODUCTION
);
```

### 4. Credential Changes
- **New credentials**: Client ID, Client Secret, Client Version
- **Old credentials**: Salt Key, Salt Index (still used for v3/v4)
- **Merchant ID**: Still required, displayed in dashboard

| Status | `/apis/pgsandbox/v1/status/{id}` | `/apis/pg/v1/status/{id}` |

## Breaking Changes to Watch

### 1. Authorization Header Format
- **v1**: `X-VERIFY: hash###index`
- **v2**: `Authorization: O-Bearer {access_token}`
- **v3/v4**: Still uses `X-VERIFY`

### 2. Request Body Structure
- **v1**: Direct JSON in body
- **v2**: Direct JSON in body (simplified)
- **v3/v4**: Base64 encoded payload in `request` field

### 3. Response Format
- **v1**: `{ success, code, data }`
- **v2**: `{ orderId, state, redirectUrl, expireAt }`
- **v3/v4**: `{ success, code, message, data }`

## SDK Updates

### Node.js SDK v2.0.5
```bash
npm i @phonepe-pg/pg-sdk-node
```

**Key Features:**
- Singleton pattern for client initialization
- Built-in callback validation
- Automatic exception handling via `PhonePeException`
- TypeScript support

**Minimum Requirements:**
- Node.js v14+

### Available Methods
```javascript
// Initiate payment
client.pay(StandardCheckoutPayRequest)

// Create SDK order
client.createSdkOrder(CreateSdkOrderRequest)

// Check status
client.getOrderStatus(merchantOrderId)

// Refund
client.refund(RefundRequest)

// Refund status
client.getRefundStatus(refundId)

// Validate callback
client.validateCallback(username, password, authHeader, responseBody)
```

## Deprecation Timeline

### What's Being Deprecated
- **v1 direct API calls** (without SDK) - still functional but not recommended
- **X-VERIFY for standard checkout** - replaced by OAuth in v2
- **Manual checksum generation** - SDK handles this internally

### What's Still Supported
- **v3/v4 APIs** - for specific use cases (UPI intent, direct debit)
- **X-VERIFY checksum** - required for v3/v4
- **Salt key authentication** - for legacy integrations

## Migration Guide: v1 to v2

### Before (v1)
```javascript
// Manual X-VERIFY generation
const hash = sha256(base64Payload + "/v1/pay" + saltKey);
const xVerify = hash + "###" + saltIndex;

// API call
fetch('https://api.phonepe.com/apis/pg/v1/pay', {
  headers: { 'X-VERIFY': xVerify },
  body: JSON.stringify({ request: base64Payload })
});
```

### After (v2)
```javascript
// SDK handles everything
const client = StandardCheckoutClient.getInstance(
  clientId, clientSecret, clientVersion, Env.PRODUCTION
);

const request = StandardCheckoutPayRequest.builder()
  .merchantOrderId(orderId)
  .amount(amount)
  .redirectUrl(redirectUrl)
  .build();

const response = await client.pay(request);
```

3. **Use SDK methods** instead of direct API calls

5. **Test thoroughly** in sandbox before production
6. **Monitor API changes** via PhonePe developer documentation
